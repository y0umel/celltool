 # NAND 阈值电压分析与读优化工具（CellTool）需求规格

  ## 1. 项目概述

  基于 WPF (.NET 8) 的桌面应用，集成 Fluent Design 风格（WPF-UI 库），用于分析 NAND Flash 在不同电压偏移下的原始读出数据，绘制阈值电压增量分布图，计算最佳读取窗口及码字纠错需求。

  ## 3. UI 布局（五选项卡）

  - 首页：快速操作区。输入/输出文件夹选择、电压范围与步长、WL 数量、起始页、开始分析按钮、进度条、实时日志窗口。
  - 数据配置页：加载厂家 Excel 表、选择芯片型号（自动填充页大小、冗余、Frame 数、WL 编码等）、GroupModel 文件管理（手动选择 csv/txt）、格雷码 MSB/LSB 配置（默认 U-M-L，全局设定）、码字长度自动计算。
  - 图表配置页：左侧属性面板（标题、轴标签、轴范围、图例文字/颜色/可见性、标注开关），右侧实时预览图表；支持保存/加载图表样式模板。
  - 自动化模式页：开关（文件监控 / HTTP API），端口配置。当前阶段预留接口桩。
  - 设置页：主题切换、全局配置文件导入导出。

  ## 4. 输入数据

  ### 4.1 电压扫描二进制文件

  - 命名：{数字}.bin 或无后缀 {数字}，数字为电压 code；1 code = 10mV
  - 布局：每个 WL 内按 Upper → Middle → Lower 页顺序连续存放
  - 每页大小 = 数据区 + 冗余区（从 Excel 获取，不区分处理）
  - 字节内 bit 序不关心（所有页同序对比）

  ### 4.2 厂家 csv 数据库

  ### 4.3 GroupModel 文本文件

  - 格式：每行一个 WL，逗号分隔 page 编号序列
  - -1 为可选占位符（表示该位置无有效 page）
  - 行数 = WL/Block
  - 示例（TLC，2 WL，6 page）：

    0,1,2,-1
    3,4,5,-1

  - 示例（SLC，2 WL，2 page）：

    0
    1

  - 由用户手动选择 csv/txt 文件

  ### 4.4 用户配置

  - 分析用 WL 数量、起始页索引
  - 电压 code 范围、code 步长
  - 格雷码 MSB/LSB 位顺序（默认 U-M-L，全局设定，UI 可切换）

  ## 5. 核心分析流水线

  ### 5.1 文件扫描与排序

  解析文件名获取电压 code，按 code 升序排列。图表横轴显示相对 code：`displayX = code - minCode`。

  ### 5.2 页数据提取

  根据 page_total_bytes（= 页数据 + 页冗余）、WL 数量、起始页及 GroupModel，用 MemoryMappedFile 从每个电压文件中切出目
  标页的字节数组。

  ### 5.3 raw Gray code 生成与物理状态映射

  - 从同一 cell 的目标 page 取对应 bit，按 MSB/LSB 配置组装为格雷码
  - GroupModel 每行有效 page 个数决定模式：1=SLC、2=MLC、3=TLC、4=QLC，并使用数据配置页对应模式的 WL编码
  - raw Gray code 用于 Vt 分布重建：源文件 raw Gray 作为 baseline；WL编码用于推导每个 `R1..Rn` 边界对应的单 page-bit 翻转与其余 page-bit 上下文
  - 需要物理状态时，再通过 WL编码序列映射为物理状态序号（0 ~ 状态数-1）

  ### 5.4 Vt 分布重建

  - 对每个电压 code V，将当前 raw Gray 与源文件 raw Gray 都映射为物理 level
  - TLC `7,6,4,0,2,3,1,5` 输出 `L0`~`L7` 八条物理源 level 曲线
  - 每条曲线统计“源文件中属于该 level 的 cell 在当前 offset 被读成其他 level”的累计错误数
  - 对累计错误数同时取递增/递减包络差分并合并，得到该 level 在左右侧 offset 下的正增量峰
  - L1..L7 的全局位置由数据配置页的 `L间距(code)` 等间距展开；默认 MLC=145、TLC=80、QLC=40
  - L1..L6 使用 `level * spacing + voltageCode`，最后一个 level 使用最后一个读边界位置；L0 保留低端擦除态位置
  - 当前主图仍使用手动 L 间距；分析结果会输出一个基于 L1..L6 相邻峰距中位数的自动建议值与置信度，仅作诊断，不自动覆盖手动设置
  - x 轴使用 code；`1 code = 10mV`
  - 无源文件时，以第一个扫压文件作为 source grouping，但结果仅视为相对/诊断曲线

  #### 5.4.1 当前错误增量实现说明

  当前生成接近目标图的实现，本质仍然是“错误 cell 数随电压变化的增量”，但不是直接比较相邻两个 offset 文件的状态变化量，而是：

  - 先用源文件/参考文件确定每个 cell 的源 raw Gray code 和物理 level
  - 对每个电压 offset 文件，将当前 raw Gray 映射为当前物理 level
  - 对每个源 level `Lx`，统计 `sourceLevel == Lx && currentLevel != sourceLevel` 的累计错误 cell 数
  - 对该累计错误曲线分别取递增包络差分和递减包络差分，再合并为正向增量峰
  - 因此图上每条 `Lx` 曲线的纵轴表示：相对源文件，该源 level 在当前电压附近新增进入/退出错误状态的 cell 数

  这种方式用源文件固定了 cell 原本属于哪个峰，再观察其随读电压偏移开始读错的位置，所以能从 offset 扫描中重建各源 level 的相对阈值分布。

  #### 5.4.2 当前峰错位限制与后续优化

  当前实现尚未真正解决“同一物理峰从左右两个读边界观测到时发生错位”的问题：

  - 当前没有按 `source -> target` 拆分左邻/右邻错误，只把 `currentLevel != sourceLevel` 归并到源 level 曲线
  - 当前没有对同一 level 的左右边界观测做互相关、质心、峰中心或最小二乘对齐
  - 左右观测峰错位较小时，合并后可能表现为更宽的单峰
  - 左右观测峰错位较大时，合并后可能出现双峰
  - `KeepDominantWindow` 目前只保留峰值最高、总量最大的主窗口，可能裁掉较弱但真实的另一侧观测
  - `AlignmentScore = null` 表示当前没有配准评分或自动重叠质量评估

  后续优化方向：

  - 按 WL 编码和页 bit 顺序建立每个读边界的 `source -> target` 左右邻居关系
  - 拆出 `L1..L7` 的左右 14 条边界观测曲线
  - 对同一物理 level 的左右观测使用互相关/质心/峰中心进行最大重叠对齐
  - 从多组可配对峰中估计 `L1-L7` 相邻间距；有效配对不足时回退到手动 L 间距
  - 输出配准 shift/score，用于判断自动对齐是否可信

  #### 5.4.3 真实分布研究记录

  真实阈值分布的基本守恒约束：每个物理 level 的分布曲线积分应接近源文件中该 level 的 cell 数；扫压范围外的 cell 需要单独列入左侧未扫到/右侧未扫到，而不是混入已观测峰内。当前研究分支新增了每个 level 的积分诊断：

  - 源Cell数：源文件中属于该 `Lx` 的 cell 数
  - 未裁剪观测积分：直接由累计错误曲线的递增/递减包络差分得到的积分
  - 显示曲线积分：当前实际画到图上的积分；当前主程序未启用主窗口裁剪，因此与未裁剪观测积分一致
  - 裁剪损失：未裁剪观测积分与显示曲线积分的差值；当前应为 0，只有重新启用主窗口显示裁剪时才非 0
  - 左侧未扫到估计：最低扫描 code 下已经读错的 cell 数
  - 右侧未扫到估计：最高扫描 code 后仍未观察到读错的 cell 数

  如果“未裁剪观测积分 + 左右未扫到”仍明显偏离源Cell数，说明当前 `sourceLevel != currentLevel` 的错误增量不是严格的守恒阈值分布。主要原因包括：同一 cell 受噪声/串扰影响在扫压中多次进入和退出错误状态；非相邻或多 bit 异常被归入同一源 level；左右边界观测被合并后失去归属；以及主窗口裁剪只改善视觉但会改变显示积分。

  后续要让结果更接近真实分布，应优先把统计口径从“源 level 读成任意其他 level”改为“按读边界拥有的单 bit 翻转事件”：

  - 根据 WL 编码拆出 `R1..Rn` 的左右邻居关系，只统计相邻边界允许的单 bit 翻转
  - 对每个 cell/每个边界只记录一次有效穿越，避免同一 cell 在噪声下反复贡献积分
  - 先生成 14 条边界侧观测曲线，再按手动 L 间距或自动配准叠加成 `L0..L7`
  - 积分校验作为质量指标输出，不应为了视觉强行归一化；只有确认漏扫范围和重复计数来源后再做校正
  - 自动对齐可用互相关、峰中心或质心，但必须输出 shift/score，低置信度时回退到手动间距

  #### 5.4.4 testdata2 参考图形态诊断

  `testdata2/Reference.png` 是非理想/磨损样例，不能只用积分判断是否贴合。当前回归测试需覆盖以下参考图特征：

  - L1 峰位在 R1=0 右侧，且 0~30 code 区间应有明显积分
  - L7 在主峰右侧保留可见拖尾，不能被窗口裁剪吞掉
  - L6 当前用源 level 累计错误包络增量、相邻绝对差分、主窗口裁剪、不同页序、以及 0 偏移源基准枚举后，峰值均约为 1100 cells；参考图视觉上可能接近或超过 1200，但没有数据口径能自然得到该峰高。后续如要求 L6>1200，必须先确认参考工具的差分定义、平滑/采样、源文件选择和归一化口径，而不能在图表层强行放大。
 
  ### 5.5 真值判定（多数投票）

  - 每个 cell 在所有扫描电压下进行多数投票
  - 平票 → 归数值更大的状态
  - 从未翻转的 cell → 视为 100% 稳定（极左或极右）

  ### 5.6 0.1% 边界

  - 期望 cell 数 = WL 总 cell 数 / 8
  - 从峰顶 code 向两侧积分增量曲线（需低通滤波预处理）
  - 累计值达到 期望cell数 × 0.1% 时记录边界 code
  - 若到达 0.1% 前进入相邻状态上升区（重叠），该侧标记为 err

  ### 5.7 最佳读电压

  - 当前单 bit 重建阶段暂不输出可靠最佳读 code
  - 后续恢复多峰识别后，在相邻峰之间的扫描 code 格点上搜索最小误判点，取谷底
  - 不插值

  ### 5.8 码字错误统计

  - 源文件对比模式：指定一个参考文件（多数投票结果或 golden file），待判断文件逐位对比
  - Page 内独立切分码字，码字长度 = page_total_bytes / Frame数（动态计算）
  - 当前阶段输出 0 偏移下的码字错误率；最佳读 code 未计算时不输出最佳读电压下码字错误率

  ## 6. 输出产物

- PNG 图像：基于源文件物理 level 错误增量重建的阈值电压分布图，TLC 输出 `L0`~`L7` 曲线，包含线性与 log 两个子图
  - CSV 报告：
      - 每条物理 level 曲线的峰位置、窗口、观测来源、配准 shift/score；最佳读 code 当前阶段仍可标记为 not calculated
      - 码字错误统计（当前阶段输出可用项；未计算最佳 code 时只输出 0 code）
      - 多 WL 时：当前阶段不输出每 WL 最佳读 code

  - 全局配置文件 (JSON)：保存/恢复 UI 所有参数、芯片选择、图表样式等

  ## 7. 自动化与外部联动（均预留接口桩）

  - HTTP API：POST /analyze（JSON 参数含文件夹路径、芯片型号 die简称、WL 数等）、GET /status/{id}、GET /result/{id}、
    DELETE /task/{id}

  - 文件监控：监控输入文件夹，新文件到达后触发分析
  - 命令行：通过 HTTP API 参数形式间接支持，不做独立 CLI 模式

  所有模式复用同一分析引擎，结果格式一致。

  ## 8. 扩展分析能力（可选输出）

  - 误码率 vs 读取电压曲线（每对相邻状态）
  - 分布拟合参数（均值、标准差、峰间距）
  - 最佳电压容限（BER 增加 10% 的电压范围）
  - 不稳定 cell 统计（投票一致性低于阈值的比例）
  - WL 间偏移统计（平均值、标准差）
  - 校准策略摘要文本建议

  ## 9. 性能与优化

  - MemoryMappedFile 按需映射，仅访问目标页区域
  - 分析任务异步执行，IProgress 更新 UI 进度，支持取消
  - 图表预览防抖更新
  - HTTP 服务与主 UI 同进程，后台线程处理

  ## 10. 错误处理

  - 文件大小校验、页大小匹配检查
  - GroupModel 行数 vs WL/Block 一致性校验
  - Excel 关键字段解析失败时提示用户手动输入
  - HTTP API 异常返回标准错误 JSON
  - 多实例互斥锁
