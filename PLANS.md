 # NAND 阈值电压分析与读优化工具（CellTool）需求规格

  ## 1. 项目概述

  基于 WPF (.NET 8) 的桌面应用，集成 Fluent Design 风格（WPF-UI 库），用于分析 NAND Flash 在不同电压偏移下的原始读出数据，绘制阈值电压增量分布图，计算最佳读取窗口及码字纠错需求。

  ## 3. UI 布局（五选项卡）

  - 首页：快速操作区。输入/输出文件夹选择、电压范围与步长、WL 数量、起始页、开始分析按钮、进度条、实时日志窗口。
  - 数据配置页：加载厂家 Excel 表、选择芯片型号（自动填充页大小、冗余、Frame 数、WL 编码等）、GroupModel 文件管理（手动选择 csv/txt）、格雷码 MSB/LSB 配置（默认 U-M-L，全局设定）、码字长度自动计算。
  - 图表配置页：左侧属性面板（标题、轴标签、轴范围、图例文字/颜色/可见性、标注开关），右侧实时预览图表；支持保存/加载图表样式模板。
  - 自动化模式页：开关（文件监控 / HTTP API），端口配置。当前阶段预留接口桩。
  - 设置页：主题切换、全局配置文件导入导出。

  ## 4. 输入数据

  ### 4.1 电压扫描二进制文件

  - 命名：{数字}.bin 或无后缀 {数字}，数字为电压 code；1 code = 10mV
  - 布局：每个 WL 内按 Upper → Middle → Lower 页顺序连续存放
  - 每页大小 = 数据区 + 冗余区（从 Excel 获取，不区分处理）
  - 字节内 bit 序不关心（所有页同序对比）

  ### 4.2 厂家 csv 数据库

  可能需要的列（选中 die 后自动填充）：

  ━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   xLC                  TLC                推断状态数、GroupModel 行有效 page 数
  ───────────────────  ─────────────────  ───────────────────────────────────────
   页 数据 Byte         16384              计算 page_total_bytes
  ───────────────────  ─────────────────  ───────────────────────────────────────

  ───────────────────  ─────────────────  ───────────────────────────────────────

  ───────────────────  ─────────────────  ───────────────────────────────────────

   WL编码               7,6,4,0,2,3,1,5    Vt 方向 raw Gray code 顺序（逗号分隔）

  解析规则：

  - 无空值、无复合值、无占位符问题（核心列均为纯数字或逗号序列）
  - 无法解析的字段提示用户手动输入
  - die简称 为下拉选项的标识符

  ### 4.3 GroupModel 文本文件

  - 格式：每行一个 WL，逗号分隔 page 编号序列
  - -1 为可选占位符（表示该位置无有效 page）
  - 行数 = WL/Block
  - 示例（TLC，2 WL，6 page）：

    0,1,2,-1
    3,4,5,-1

  - 示例（SLC，2 WL，2 page）：

    0
    1

  - 由用户手动选择 csv/txt 文件

  ### 4.4 用户配置

  - 分析用 WL 数量、起始页索引
  - 电压 code 范围、code 步长
  - 格雷码 MSB/LSB 位顺序（默认 U-M-L，全局设定，UI 可切换）

  ## 5. 核心分析流水线

  ### 5.1 文件扫描与排序

  解析文件名获取电压 code，按 code 升序排列。图表横轴显示相对 code：`displayX = code - minCode`。

  ### 5.2 页数据提取

  根据 page_total_bytes（= 页数据 + 页冗余）、WL 数量、起始页及 GroupModel，用 MemoryMappedFile 从每个电压文件中切出目
  标页的字节数组。

  ### 5.3 raw Gray code 生成与物理状态映射

  - 从同一 cell 的目标 page 取对应 bit，按 MSB/LSB 配置组装为格雷码
  - GroupModel 每行有效 page 个数决定模式：1=SLC、2=MLC、3=TLC、4=QLC，并使用数据配置页对应模式的 WL编码
  - raw Gray code 用于增量分布：当前文件与上一文件 raw Gray code 是否变化
  - 需要物理状态时，再通过 WL编码序列映射为物理状态序号（0 ~ 状态数-1）

  ### 5.4 增量分布计算

  - 对每个电压 code V，只绘制一条总增量曲线
  - 曲线纵轴为当前文件相对上一文件 raw Gray code 发生变化的 cell 数
  - 首个文件与全 `0xFF` 空白页组出的 raw Gray code 对比
  - 当前阶段不按 `WL编码` 的相邻转移拆分多条曲线；任何 raw Gray code 差异都计入总变化
  - 后续优化：根据当前电压前已有几个峰，推断该电压区域理论上只应发生哪一种 Gray 翻转，再过滤非预期跳变以降低串扰干扰

  ### 5.5 真值判定（多数投票）

  - 每个 cell 在所有扫描电压下进行多数投票
  - 平票 → 归数值更大的状态
  - 从未翻转的 cell → 视为 100% 稳定（极左或极右）

  ### 5.6 0.1% 边界

  - 期望 cell 数 = WL 总 cell 数 / 8
  - 从峰顶 code 向两侧积分增量曲线（需低通滤波预处理）
  - 累计值达到 期望cell数 × 0.1% 时记录边界 code
  - 若到达 0.1% 前进入相邻状态上升区（重叠），该侧标记为 err

  ### 5.7 最佳读电压

  - 单条总增量曲线阶段暂不输出可靠最佳读 code
  - 后续恢复多峰识别后，在相邻峰之间的扫描 code 格点上搜索最小误判点，取谷底
  - 不插值

  ### 5.8 码字错误统计

  - 源文件对比模式：指定一个参考文件（多数投票结果或 golden file），待判断文件逐位对比
  - Page 内独立切分码字，码字长度 = page_total_bytes / Frame数（动态计算）
  - 当前阶段输出 0 偏移下的码字错误率；最佳读 code 未计算时不输出最佳读电压下码字错误率

  ## 6. 输出产物

  - PNG 图像：阈值电压增量分布图，当前阶段只含一条 total raw Gray changes 曲线、图例与自定义轴标题
  - CSV 报告：
      - 单条总增量曲线参数；峰参数和最佳读 code 当前阶段标记为 not calculated
      - 码字错误统计（当前阶段输出可用项；未计算最佳 code 时只输出 0 code）
      - 多 WL 时：当前阶段不输出每 WL 最佳读 code

  - 全局配置文件 (JSON)：保存/恢复 UI 所有参数、芯片选择、图表样式等

  ## 7. 自动化与外部联动（均预留接口桩）

  - HTTP API：POST /analyze（JSON 参数含文件夹路径、芯片型号 die简称、WL 数等）、GET /status/{id}、GET /result/{id}、
    DELETE /task/{id}

  - 文件监控：监控输入文件夹，新文件到达后触发分析
  - 命令行：通过 HTTP API 参数形式间接支持，不做独立 CLI 模式

  所有模式复用同一分析引擎，结果格式一致。

  ## 8. 扩展分析能力（可选输出）

  - 误码率 vs 读取电压曲线（每对相邻状态）
  - 分布拟合参数（均值、标准差、峰间距）
  - 最佳电压容限（BER 增加 10% 的电压范围）
  - 不稳定 cell 统计（投票一致性低于阈值的比例）
  - WL 间偏移统计（平均值、标准差）
  - 校准策略摘要文本建议

  ## 9. 性能与优化

  - MemoryMappedFile 按需映射，仅访问目标页区域
  - 分析任务异步执行，IProgress 更新 UI 进度，支持取消
  - 图表预览防抖更新
  - HTTP 服务与主 UI 同进程，后台线程处理

  ## 10. 错误处理

  - 文件大小校验、页大小匹配检查
  - GroupModel 行数 vs WL/Block 一致性校验
  - Excel 关键字段解析失败时提示用户手动输入
  - HTTP API 异常返回标准错误 JSON
  - 多实例互斥锁
